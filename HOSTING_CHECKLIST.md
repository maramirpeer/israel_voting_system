# קול משותף - חיבור אחסון, נתונים ומייל

## מה צריך להיות בשליטת אמיר

- חשבון הדומיין `sharedemocracy.com`.
- חשבון האחסון שבו רץ האתר.
- מסד הנתונים של האתר.
- שירות שליחת המיילים.
- גיבוי קבוע של טפסי ההצטרפות.

אפשר לתת לחבר הרשאת עזרה, אבל החשבונות והסיסמאות הראשיות צריכים להישאר אצל אמיר.

## משתני סביבה לשרת

להגדיר ב-Host:

```env
DATABASE_URL=mysql://USER:PASSWORD@HOST:3306/DATABASE_NAME
ADMIN_SIGNUPS_TOKEN=מפתח ארוך וסודי למסך הניהול
RESEND_API_KEY=המפתח של Resend
MEMBER_SIGNUP_EMAIL_FROM=קול משותף <noreply@sharedemocracy.com>
```

אם אין `DATABASE_URL` בפרודקשן, האתר לא יקבל טפסי הצטרפות ולא יציג רשומות בניהול. קובץ מקומי מתאים רק לפיתוח מקומי, או להפעלה מפורשת עם `ALLOW_LOCAL_SIGNUP_STORE=true`.

## טבלת טפסי ההצטרפות

צריך להריץ במסד הנתונים את הקובץ:

```text
drizzle/0008_member_signups.sql
```

הטבלה שומרת:

- שם מלא
- תעודת זהות
- אימייל
- טלפון
- הערה
- תאריך יצירה ועדכון

האתר מציג לציבור רק שמות מצטרפים, ולא מציג תעודות זהות, טלפונים או אימיילים.

## מיילים

האתר שולח מייל ברכה אחרי הרשמה דרך Resend. כדי שזה יעבוד:

- לפתוח חשבון Resend בבעלות אמיר.
- לאמת את הדומיין `sharedemocracy.com`.
- להוסיף בדומיין את רשומות ה-DNS ש-Resend נותן.
- להכניס את `RESEND_API_KEY` לשרת בלבד.

לא להעלות API keys או סיסמאות ל-GitHub.

## גיבוי

לבקש מה-Host:

- גיבוי אוטומטי יומי למסד הנתונים.
- אפשרות יצוא CSV/JSON של כל טפסי ההצטרפות.
- הרשאת גישה מלאה לאמיר למסד הנתונים.
